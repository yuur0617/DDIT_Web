DOM(Document Object Model, 문서 객체 모델)
    - 브라우저가 웹 문서를 이해하도록 구성된 문서 객체 모델
    - Document와 관련된 객체들의 집합
    - 프로그래밍 언어가 DOM구조에 접근할 수 있는 방법 제공 >> 구조, 스타일, 내용 변경 가능
    - 브라우저는 웹 문서의 요소들을 객체화하고 구조화하여  
        상/하위 또는 병렬구조로 체계화 함(= 트리구조, 부자관계 형성)

    [DOM Tree 구조]
    Document(문서 노드) : 문서 전체를 나타내는 트리 최상단, 하위로 접근하기 위한 시작점
      ★ DocumentType
      ┗★ Elemnet(요소 노드) : 각종 태그들
          ┗★Attribute(속성노드)
          ┗ Entity
          ┗ EntityReference
          ┗ Text(텍스트 노드) : 요소 노드의 자식이자 최종단 
      ┗ Comment(주석 노드)
      ┗ CDATASection
      ┗ Notation
디브안에 스팬1 디브안에 스팬2 디브안에 스팬3 
    [Node간 관계 종류]
    - node대상 속성(elementNode 대상 속성)
    - parentNode(parentElement)
    - childNodes(children)
    - firstChild(firstElementChild)
    - lastChild(lastElementChild)
    - nextSibling(nextElementSibling)
    - previousSibling(previousElementSibling)

    [Node정보]
    - nodeName >> 노드 고유의 이름 / 읽기전용 속성(get만 가능 / set은 불가능)
    - nodevalue >> 노드의 값 - text를 자식으로 갖는 textNode 
    - nodeType >> 노드 고유의 타입을 정수형태로 반환 / 읽기전용 속성

    [Node 생성 및 추가]
    - createElement()
    - createAttribute()
    - createTextNode()

    - append() - node대상과 DOMString(문자열요소, 파싱되지 않고 문자 자체로 표현) 둘다 추가
    - appendChild() = node대상으로만 추가, 대상요소의 마지막에 추가
    - insertBefore(node, 삽입할 위치) - 삽입위치를 null로 지정할 경우 appendChild와 동일(마지막에 추가)
    - insetData(offset,data) - 텍스트 노드의 offset(default 0, 삽입 위치)에 새 텍스트 추가

    [Node삭제, 복사]
    - removechild()
    - remove
    - cloneNode()
